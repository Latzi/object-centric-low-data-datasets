# Cityscapes--Pedestrian Annotations

This folder documents the annotation layer for the **Cityscapes--Pedestrian** subset.

## Purpose

Because this subset is released in **abstract / non-image form**, annotation artifacts released here are limited to non-image metadata and derived annotation information that does not redistribute Cityscapes image content.

The annotation layer makes the Cityscapes--Pedestrian subset inspectable and reusable without distributing Cityscapes-derived image pixels.

## Public annotation table

This folder includes the canonical public annotation table for the Cityscapes--Pedestrian subset:

- `cityscapes_pedestrian_boxes.csv`

This CSV contains one row per pedestrian-related bounding box associated with the released public manifest records.

The table does **not** contain:

- image pixels
- cropped pedestrian images
- thumbnails
- preview images
- mask images derived from Cityscapes crops
- any image-derived artifact intended to substitute for the Cityscapes image subset

The annotation table is intended to support:

- annotation inspection
- split-level statistics
- reproducibility checks
- bounding-box mask rasterization
- reconstruction workflows for authorized users
- consistency checks against the public manifest
- analysis of object density and multi-box structure

The annotation table should be interpreted together with:

- `../manifests/cityscapes_pedestrian_manifest.csv`
- `../metadata/cityscapes_pedestrian_summary.json`
- `../metadata/pipeline_config_cityscapes.json`

## Annotation table format

The public annotation table uses one row per bounding box.

Expected columns include:

- `dataset`
- `sample_id`
- `source_image_id`
- `split`
- `box_index`
- `class_id`
- `class_name`
- `x_center`
- `y_center`
- `width`
- `height`
- `format`
- `crop_size`
- `notes`

Bounding-box coordinates are stored in YOLO-normalized format:

```text
x_center y_center width height
```

where coordinates are normalized to the reconstructed crop size.

## Intended annotation role

The annotation layer is intended to support:

- class mapping
- released sample identifiers
- split-aware annotation lookup
- derived non-image artifact tracking
- consistency with the public manifest and metadata files
- bounding-box-mask generation
- lightweight dataset statistics

## Class definition

The Cityscapes--Pedestrian subset uses a merged pedestrian class derived from pedestrian-related categories in the upstream dataset.

Public release notation for this subset should use:

```text
0 -> pedestrian
```

## Relationship to masks

Explicit binary mask image files are not distributed in this repository.

If a model requires bounding-box masks, users can rasterize binary masks from the bounding-box rows in:

```text
cityscapes_pedestrian_boxes.csv
```

For a `256x256` reconstructed crop, a bounding-box mask can be generated by setting pixels inside the bounding box to `1` and pixels outside the box to `0`.
