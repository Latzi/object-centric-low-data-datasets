# COCO PottedPlant Annotations

This folder documents the annotation layer for the **COCO PottedPlant** subset.

## Purpose

Because this subset is released in **metadata / reconstruction-first form**, the annotation layer in this repository is primarily a public non-image annotation artifact rather than a blanket packaged release of upstream-derived image files.

The annotation layer makes the COCO PottedPlant subset inspectable and reusable without assuming blanket redistribution rights for all COCO-derived cropped images.

## Public annotation table

This folder includes the canonical public annotation table for the COCO PottedPlant subset:

- `coco_pottedplant_boxes.csv`

This CSV contains one row per potted-plant bounding box in the locally reconstructed cropped YOLO subset.

The table does **not** contain:

- image pixels
- cropped COCO images
- thumbnails
- preview images
- mask images derived from COCO crops
- any image-derived artifact intended to substitute for the reconstructed COCO crop subset

The annotation table is intended to support:

- annotation inspection
- split-level statistics
- reproducibility checks
- bounding-box mask rasterization
- reconstruction workflows for users who obtain COCO separately
- consistency checks against the public manifest
- analysis of object density and multi-box structure

The annotation table should be interpreted together with:

- `../manifests/coco_pottedplant_manifest.csv`
- `../metadata/coco_pottedplant_summary.json`
- `../metadata/pipeline_config_coco.json`

## Annotation flow in the reconstruction pipeline

The current COCO pipeline has two annotation-producing stages.

### Step 01 — Full-image YOLO subset

`01_extract_pottedplant_from_coco_to_yolo.py` reads the official COCO images and annotations, selects the target category, and writes a YOLO-format full-image subset that preserves the original COCO split structure.

Expected local outputs include:

```text
YOLO_pottedplant/labels/train/
YOLO_pottedplant/labels/val/
```

### Step 02 — Cropped YOLO subset

`02_create_instance_crops_from_yolo_pottedplant.py` creates `256x256` per-instance crops from the Step 01 subset and writes cropped YOLO labels for those crops.

Expected local outputs include:

```text
YOLO_pottedplant_cropped/labels/train/
YOLO_pottedplant_cropped/labels/val/
```

The current crop-generation configuration keeps all intersecting potted-plant boxes in the cropped labels so that visible additional potted plants are not left unlabeled.

## Annotation table format

The public annotation table uses one row per bounding box.

Expected columns include:

- `dataset`
- `sample_id`
- `source_image_id`
- `split`
- `box_index`
- `class_id`
- `class_name`
- `x_center`
- `y_center`
- `width`
- `height`
- `format`
- `crop_size`
- `notes`

Bounding-box coordinates are stored in YOLO-normalized format:

```text
x_center y_center width height
```

where coordinates are normalized to the reconstructed crop size.

## Class mapping

The public standardized class name for this subset is:

```text
0 -> potted_plant
```

## Relationship to masks

Explicit binary mask image files are not distributed in this repository.

If a model requires bounding-box masks, users can rasterize binary masks from the bounding-box rows in:

```text
coco_pottedplant_boxes.csv
```

For a `256x256` reconstructed crop, a bounding-box mask can be generated by setting pixels inside the bounding box to `1` and pixels outside the box to `0`.
